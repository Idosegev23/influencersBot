עקרונות מנחים

Multi-tenant מ-day 1

כל טבלת domain מכילה account_id. RLS policies מסננות לפי auth.uid() שממופה ל-account. SUPABASE_SERVICE_ROLE_KEY עוקף RLS לצרכי שרת.

RAG-first

כל תוכן עובר chunking + embedding + indexing. retrieval משלב vector + BM25 + grounded_facts.

Streaming everywhere

SSE בצ'אט, after() ל-side effects (CAPI, Drive), אסור לעצור את הזרם בשביל פעולות איטיות.

Defensive trimming

env vars מ-Vercel נוטים להגיע עם \n בסוף. תמיד .trim() בקריאה.

RAG Retrieval Algorithm (5 Steps · retrieve.ts)

  1. Step 0: Query expansion via Gemini 2.5 Flash (גישור פערי vocabulary)
  2. Step 1: Classification - structured (counts/dates) vs unstructured. inference של entity types & topics. time hints
  3. Step 2: Filtering - hard filters: account, entity type, time window, metadata
  4. Step 3: Vector + BM25 Hybrid - threshold 0.3, fallback 0.25. אם top similarity <0.6 → expand + re-search. BM25 רק כש-vector לא בטוח >0.78. merge + dedup
  5. Step 4: Heuristic Reranking - recency bonus (+0.05 ל-30 ימים), first chunk (+0.03), exact phrase (+0.25), partial keyword (+0.08-0.15), topic compatibility, LDRS conference boost
  6. Step 5: Build Context - sort + filter (<0.35), dedup per source, format כ-XML ל-LLM

Chunker

  • Target: 400 tokens per chunk (min 100, max 500)
  • Overlap: 12% (48 tokens על chunk של 400)
  • Token estimation: length / 3.5 (3 chars/token לעברית)
  • Smart split: paragraph → newline → sentence → clause → word → hard cut
  • Noise filter: skips nutritional tables & metadata-only sections

Security & Compliance

איך המערכת שומרת על נתונים, על אילו threats מגנה, ומה ה-compliance posture.

שכבות אבטחה

1. Network

HTTPS-only · HSTS · CSP · X-Frame-Options · Permissions-Policy. הגדרות ב-next.config.ts.

2. Application

Rate limiting · sanitize input · output encoding · Auth middleware על routes רגישים.

3. Database

Postgres RLS על כל טבלת domain · multi-tenant isolation · service-role בלעדי לשרת.

4. Secrets

Vercel env (encrypted) · אסור לקמיט .env* · API keys מתחלפים תקופתית.

Streaming + after()

כל route ארוך משתמש ב-after() של Next 16 כדי לחזיר response מהר ולהמשיך side effects ברקע (Drive upload, CAPI, email):

import { after } from 'next/server';

export async function POST(req: Request) {
  const result = await processFast();

  after(async () => {
    await uploadToDrive(result);  // doesn't block response
    await fireCAPI(result);
    await sendConfirmationEmail(result);
  });

  return Response.json(result); // returned immediately
}
